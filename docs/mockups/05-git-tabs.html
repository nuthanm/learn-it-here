GIT

Version control, demystified.

Branching strategies that scale

Branches are cheap in Git. The real cost is coordination — keep branches short-lived and merge often.

Trunk-based

Everyone commits to main behind feature flags. Best for teams shipping multiple times a day. 

$ git switch -c feat/checkout-redesign
$ git push -u origin feat/checkout-redesign
# open PR → merge within 24h
💡 Tip: name branches type/short-description — your future self will thank you.

GitFlow

Heavier, with develop, release/*, and hotfix/* branches. Useful for versioned products with slow release cadences.

⚠️ Gotcha: long-lived branches drift. Rebase or merge from main at least daily.

Blazor vs Web Forms

A side-by-side look at where each shines — and where it doesn't.

Blazor Web Forms
Rendering model Component-based Page lifecycle + postback
Language C# (Razor) C# / VB (.aspx)
State management Scoped DI, services ViewState (heavy)
Routing Attribute / file-based File-system based
Tooling VS, VS Code, Rider VS only
Hosting Server, WASM, Hybrid IIS only
Active development Yes — first-class Maintenance mode

✅ Pick Blazor for new work

Modern component model, cross-platform tooling, and a clear roadmap. Best fit for greenfield apps and modernization efforts.

🧓 Keep Web Forms only when needed

Reasonable for maintaining existing line-of-business apps where a rewrite isn't justified. Plan a migration path.
